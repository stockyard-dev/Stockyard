The problem

You shipped an app with LLM calls. Now you need cost caps, caching, safety filters, routing, observability, prompt management, workflow orchestration, and a way to share configs across environments. You're not going to set up six different tools with Redis and Postgres just to stop surprise bills.

The fix

Stockyard is one Go binary that gives you six integrated apps. Point your OPENAI_BASE_URL at it and you get a full platform — proxy with 50+ middleware modules, real-time observability, trust and audit trails, prompt studio, DAG workflow engine, and a config marketplace. Works with OpenAI, Anthropic, Gemini, Groq, Ollama, and 12 more providers.

The platform
App 01 Proxy The gateway layer. 50+ middleware modules, runtime toggles. Caching • Rate limiting • Cost caps • Failover • PII redaction • Content filter • Retry • Load balancing • Circuit breakers
App 02 Observe See everything. Every request, every dollar, every anomaly. Request tracing • Cost rollups • Latency tracking • Alert rules • Anomaly detection • Daily dashboards
App 03 Trust Append-only audit ledger. Hash-chained. Tamper-evident. Audit trails • Evidence packs • Policy engine • Feedback capture • Compliance reports • Replay
App 04 Studio Prompt engineering, versioned. Experiment, benchmark, snapshot. Prompt templates • A/B experiments • Benchmarks • Snapshot comparison • Version history
App 05 Forge DAG workflow engine. Chain LLM calls with dependencies. Topological execution • Template variables • Transform steps • Tool integration • Batch processing
App 06 Exchange Config packs. Install a stack in one click. Starter packs • One-click install • Clean uninstall • Environment sync • Version control
How it works

Three commands. Thirty seconds.

# Install curl -sSL stockyard.dev/install | sh # Run (starts all 6 apps on :4200 with console at /ui) stockyard # Point your app at the proxy export OPENAI_BASE_URL=http://localhost:4200/v1 # That's it. Every LLM call now flows through the platform. # Open http://localhost:4200/ui for the live console.
