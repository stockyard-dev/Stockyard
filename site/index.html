The problem

You shipped an app with LLM calls. Now you've got surprise bills, flaky providers, no caching, no rate limits, no visibility into what's happening. You need infrastructure, but you're one person with a weekend project — you're not going to set up Kubernetes and Redis just to add a spending cap.

The fix

Stockyard sits between your app and the LLM provider. One binary, one config file. Point your OPENAI_BASE_URL at it and you get spending caps, caching, fallback routing, rate limiting, logging, and 120 more tools — all without changing a line of application code. It works with OpenAI, Anthropic, Gemini, Groq, Ollama, and 12 more providers.

How it works

Three commands. Thirty seconds.

# Install curl -sSL stockyard.dev/install | sh # Run (starts proxy on :4000 with dashboard at /ui) stockyard --config stockyard.yaml # Point your app at the proxy export OPENAI_BASE_URL=http://localhost:4000/v1 # That's it. Every LLM call now flows through Stockyard. # Open http://localhost:4000/ui for the live dashboard.
