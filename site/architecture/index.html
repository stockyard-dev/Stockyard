How it works

Every request enters through the standard OpenAI-compatible /v1/chat/completions endpoint. It flows through the middleware chain in order: routing decides which provider handles it, safety modules check for harmful content, cost modules enforce spending limits, the cache checks for a hit, transforms optimize the prompt, validators ensure output quality, and observe modules record everything.

After the middleware chain, the request hits the selected provider. On the way back, the response flows through the chain in reverse. Observe automatically writes a trace record and Trust writes a hash-chained audit entry — no configuration needed.

Every module is wrapped with toggle.Wrap. Disabled modules are bypassed with zero overhead. Toggle any module on or off at runtime via the API or the console — no restart required.
